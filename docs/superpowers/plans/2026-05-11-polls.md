# HeyFam Polls v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the canonical "what's for dinner?" specialization — single-choice family polls that live in the feed alongside regular posts, with live result reveal, optional close-by date, and optional anonymous voting. Posting a poll uses the existing post fan-out; casting a vote is silent.

**Architecture:** A poll is a regular WP post decorated with three `post_meta` keys (`heyfam_poll_options`, `heyfam_poll_closes_at`, `heyfam_poll_anon`) — the post body is the poll question. Votes live in a per-subsite custom table `wp_{N}_heyfam_poll_votes` with `UNIQUE (post_id, user_id)` so each fam member has exactly one vote per poll (changeable via `INSERT … ON DUPLICATE KEY UPDATE`). `Routes::serialize_post()` adds a `poll` field (null for regular posts; a structured object with options, totals, the caller's vote, voters-by-option, and a `closed` flag for polls). The existing `state.posts` array does double-duty — the post-card template branches on `context.item.poll` to render the poll widget instead of the body. SSR via `wp_interactivity_state` means polls don't flicker on first paint. A new `POST /{fam}/poll-vote` casts a vote and the feed refreshes via the existing 10s poll cycle plus an immediate `heyfam/feed::callbacks.refresh` kick.

**Tech Stack:** PHP 8.1+, WordPress 6.5+ multisite, `@wordpress/interactivity` (IAPI), esbuild, native browser web push (out of scope here).

**Depends on:** **design-system** (Phase B), **media** (Phase D second). Lands **last** in Phase D. The composer.html, post-card.html, composer.js, and `serialize_post` edits in this plan are additive — they patch on top of the design-system baseline and (where the keys/markup overlap) on top of media's edits.

## Design decisions

These resolve the open questions in `docs/superpowers/orchestrator.md` Section 6 and in the task brief.

| Question | Decision | Rationale |
|---|---|---|
| Post type vs `post_meta` | `post_meta` on a regular `post` | Feed shape stays intact; polls just decorate certain posts. Existing post-card, single-page, reactions, comments, and FanOut all keep working unmodified for polls. |
| Multi-choice? | Single-choice for v1 | Hobby scope. Multi-select is a v2 widening of the UNIQUE key + extra UI state. |
| Compose UI | Toggle button in the composer that swaps the body textarea + photo button for a question textarea + options list (2–6 inputs) + close-date + anon checkbox | No modal step; no separate compose surface. Same form, same submit endpoint. |
| Vote UI | Tap an option to vote. Result bars are always visible — empty fill pre-any-vote, fills in with count + percentage as votes land. Your option gets the accent color. | "Always show results" matches the read of a family poll — there's nothing to hide; transparency reduces friction. |
| Server endpoint for create | Extend `POST /{fam}/posts` to accept optional `question`, `options[]`, `closes_at`, `anon` | One endpoint, one fan-out hook, no auth split. |
| Server endpoint for vote | New `POST /{fam}/poll-vote` (with `option_index`, `post_id`) | Distinct verb. Voting doesn't change post fields; treat it as a separate target. |
| Close-by date | Optional `closes_at` ISO datetime. After it passes, voting returns `closed`. UI disables buttons + shows a small "closed" badge. | Light-touch surface; one DB column comparison server-side. |
| Anonymous votes | Per-poll setting (composer checkbox). Default: revealed within fam. | Recommended. Voter names are visible to the fam (small group, the original brief assumes trust), but the creator can opt anonymous when needed (e.g. "are we cancelling Christmas?"). |
| Reactions on polls | Yes — same affordance as posts | Polls are decorated posts. No special-casing. |
| Comments on polls | Yes — same affordance as posts | Same reasoning. |
| Notifications | Posting a poll fans out as a regular post (existing hook). Voting does NOT fan out. | Voting is high-frequency; notification noise would be unbearable. The poll author sees results when they reopen the post. |
| Change your vote? | Yes — `INSERT … ON DUPLICATE KEY UPDATE` semantics | Hobby scope, forgiving UX. Tapping a different option replaces; no separate confirmation step. |
| SSR vote state | Yes — `Routes::serialize_post()` includes `my_vote` for the current user so the chosen option is highlighted on first paint. | Avoids flicker on hydration. Matches the existing post serialization pattern. |

## Data model

### `wp_{N}_heyfam_poll_votes` (per subsite)

```sql
CREATE TABLE wp_{N}_heyfam_poll_votes (
  id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  post_id BIGINT UNSIGNED NOT NULL,
  user_id BIGINT UNSIGNED NOT NULL,
  option_index TINYINT UNSIGNED NOT NULL,
  voted_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  UNIQUE KEY uniq_post_user (post_id, user_id),
  KEY post_idx (post_id)
) <CHARSET>;
```

`UNIQUE (post_id, user_id)` is what enforces single-choice. Changing a vote uses `INSERT … ON DUPLICATE KEY UPDATE option_index = VALUES(option_index), voted_at = NOW()`.

### `post_meta`

- `heyfam_poll_options` — JSON-encoded array of strings, 2 ≤ count ≤ 6. Sanitized + length-capped.
- `heyfam_poll_closes_at` — MySQL UTC datetime, or empty string for "never closes".
- `heyfam_poll_anon` — `'1'` or `'0'`.

A post is a poll iff `heyfam_poll_options` exists and decodes to an array of length ≥ 2.

### Serialized `poll` shape (in `serialize_post`)

```php
[
  'options'      => [ 'Pizza', 'Sushi', 'Tacos' ],      // strings
  'closes_at'    => null,                                // ISO 8601 string or null
  'closed'       => false,                               // server-computed: closes_at && now > closes_at
  'anon'         => false,
  'total_votes'  => 5,
  'results'      => [ 2, 1, 2 ],                         // counts aligned with options indices
  'percents'     => [ 40, 20, 40 ],                      // integer percentages aligned with options
  'my_vote'      => 0,                                   // option index, or -1 if not voted
  'voters'       => [                                    // empty array when anon=true OR when there are no votes
    [ 'name' => 'Alice', 'avatar_url' => 'data:image/svg+xml;base64,...', 'option_index' => 0 ],
    ...
  ],
]
```

For non-poll posts, `poll` is `null`.

- [ ] **Step 3: Create `plugins/heyfam-core/src/Polls/Manager.php`**

```php
<?php
namespace HeyFam\Core\Polls;

final class Manager {
    public static function table_name( ?int $blog_id = null ): string {
        global $wpdb;
        $bid = $blog_id ?? get_current_blog_id();
        return $wpdb->base_prefix . $bid . '_heyfam_poll_votes';
    }

    public static function create_table( int $blog_id ): void {
        global $wpdb;
        $table   = self::table_name( $blog_id );
        $charset = $wpdb->get_charset_collate();
        require_once ABSPATH . 'wp-admin/includes/upgrade.php';
        dbDelta( "CREATE TABLE $table (
            id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
            post_id BIGINT UNSIGNED NOT NULL,
            user_id BIGINT UNSIGNED NOT NULL,
            option_index TINYINT UNSIGNED NOT NULL,
            voted_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
            PRIMARY KEY (id),
            UNIQUE KEY uniq_post_user (post_id, user_id),
            KEY post_idx (post_id)
        ) $charset;" );
    }

    /**
     * Returns null on success, or an error code: 'bad_option' | 'closed' | 'not_a_poll'.
     */
    public static function vote( int $post_id, int $user_id, int $option_index ): ?string {
        $options = self::options_for( $post_id );
        if ( ! $options ) return 'not_a_poll';
        if ( $option_index < 0 || $option_index >= count( $options ) ) return 'bad_option';
        if ( self::is_closed( $post_id ) ) return 'closed';

        global $wpdb;
        $table = self::table_name();
        $wpdb->query( $wpdb->prepare(
            "INSERT INTO $table (post_id, user_id, option_index)
             VALUES (%d, %d, %d)
             ON DUPLICATE KEY UPDATE option_index = VALUES(option_index), voted_at = CURRENT_TIMESTAMP",
            $post_id, $user_id, $option_index
        ) );
        return null;
    }

    public static function counts_for( int $post_id, int $option_count ): array {
        global $wpdb;
        $table = self::table_name();
        $rows  = $wpdb->get_results( $wpdb->prepare(
            "SELECT option_index, COUNT(*) AS n FROM $table WHERE post_id = %d GROUP BY option_index",
            $post_id
        ), ARRAY_A );
        $out = array_fill( 0, $option_count, 0 );
        foreach ( $rows as $r ) {
            $i = (int) $r['option_index'];
            if ( $i >= 0 && $i < $option_count ) $out[ $i ] = (int) $r['n'];
        }
        return $out;
    }

    /** Returns option_index for $user_id on $post_id, or -1 if no vote. */
    public static function my_vote( int $post_id, int $user_id ): int {
        if ( ! $user_id ) return -1;
        global $wpdb;
        $table = self::table_name();
        $row = $wpdb->get_var( $wpdb->prepare(
            "SELECT option_index FROM $table WHERE post_id = %d AND user_id = %d",
            $post_id, $user_id
        ) );
        return $row === null ? -1 : (int) $row;
    }

    /** Returns voters keyed by option_index, with display info for the IAPI template. */
    public static function voters_for( int $post_id ): array {
        global $wpdb;
        $table = self::table_name();
        $rows  = $wpdb->get_results( $wpdb->prepare(
            "SELECT user_id, option_index FROM $table WHERE post_id = %d ORDER BY voted_at ASC",
            $post_id
        ), ARRAY_A );
        $out = [];
        foreach ( $rows as $r ) {
            $u = get_userdata( (int) $r['user_id'] );
            if ( ! $u ) continue;
            $name = $u->display_name ?: $u->user_login;
            $out[] = [
                'name'         => $name,
                // design-system's Avatar helper returns either an uploaded image URL or a
                // deterministic boring-avatars data: URI. Matches the contract on
                // serialize_post/serialize_comment.
                'avatar_url'   => \HeyFam\Core\Avatars\Avatar::url_for_user( (int) $u->ID ),
                'option_index' => (int) $r['option_index'],
            ];
        }
        return $out;
    }

    /** Decoded options array (≥2 strings), or null if this post isn't a poll. */
    public static function options_for( int $post_id ): ?array {
        $raw = get_post_meta( $post_id, 'heyfam_poll_options', true );
        if ( ! is_string( $raw ) || $raw === '' ) return null;
        $decoded = json_decode( $raw, true );
        if ( ! is_array( $decoded ) || count( $decoded ) < 2 ) return null;
        return array_values( array_map( 'strval', $decoded ) );
    }

    public static function is_closed( int $post_id ): bool {
        $closes = (string) get_post_meta( $post_id, 'heyfam_poll_closes_at', true );
        if ( $closes === '' ) return false;
        $ts = strtotime( $closes . ' UTC' );
        return $ts && $ts < time();
    }

    public static function is_anon( int $post_id ): bool {
        return (string) get_post_meta( $post_id, 'heyfam_poll_anon', true ) === '1';
    }
}
```

- [ ] **Step 1: Extend `Composer::create()` to accept poll fields**

Replace the body of `plugins/heyfam-core/src/Posts/Composer.php` with:

```php
<?php
namespace HeyFam\Core\Posts;

final class Composer {
    /**
     * Create a post in the given fam (already-switched blog context).
     * $photo: array from $_FILES (or null).
     * $poll:  optional array with keys: options[], closes_at (str|null), anon (bool).
     *         When present, the resulting post is a poll: body is the question.
     */
    public static function create(
        int $author_id,
        string $body,
        ?array $photo = null,
        ?array $poll = null
    ): int|\WP_Error {
        $body = trim( $body );
        $is_poll = $poll !== null;

        if ( $is_poll ) {
            $opts = self::sanitize_options( $poll['options'] ?? [] );
            if ( count( $opts ) < 2 ) {
                return new \WP_Error( 'too_few_options', 'A poll needs at least two options.' );
            }
            if ( count( $opts ) > 6 ) {
                return new \WP_Error( 'too_many_options', 'A poll can have up to 6 options.' );
            }
            if ( $body === '' ) {
                return new \WP_Error( 'no_question', 'Polls need a question.' );
            }
        } elseif ( $body === '' && ! $photo ) {
            return new \WP_Error( 'empty_post', 'Add some text or a photo.' );
        }

        $post_id = wp_insert_post( [
            'post_type'    => 'post',
            'post_status'  => 'publish',
            'post_author'  => $author_id,
            'post_content' => wp_kses_post( $body ),
        ], true );
        if ( is_wp_error( $post_id ) ) return $post_id;

        if ( $is_poll ) {
            update_post_meta( $post_id, 'heyfam_poll_options', wp_json_encode( $opts ) );

            $closes_at = isset( $poll['closes_at'] ) ? (string) $poll['closes_at'] : '';
            $closes_mysql = '';
            if ( $closes_at !== '' ) {
                $ts = strtotime( $closes_at );
                if ( $ts && $ts > time() ) {
                    $closes_mysql = gmdate( 'Y-m-d H:i:s', $ts );
                }
            }
            update_post_meta( $post_id, 'heyfam_poll_closes_at', $closes_mysql );

            $anon = ! empty( $poll['anon'] );
            update_post_meta( $post_id, 'heyfam_poll_anon', $anon ? '1' : '0' );
        }

        if ( $photo && ! empty( $photo['tmp_name'] ) ) {
            require_once ABSPATH . 'wp-admin/includes/file.php';
            require_once ABSPATH . 'wp-admin/includes/media.php';
            require_once ABSPATH . 'wp-admin/includes/image.php';

            $attach_id = media_handle_sideload( $photo, $post_id );
            if ( is_wp_error( $attach_id ) ) {
                error_log( '[heyfam] photo upload failed: ' . $attach_id->get_error_message() );
            } else {
                set_post_thumbnail( $post_id, $attach_id );
            }
        }

        return (int) $post_id;
    }

    private static function sanitize_options( array $raw ): array {
        $out = [];
        foreach ( $raw as $o ) {
            $s = trim( wp_strip_all_tags( (string) $o ) );
            if ( $s === '' ) continue;
            if ( mb_strlen( $s ) > 80 ) $s = mb_substr( $s, 0, 80 );
            $out[] = $s;
        }
        return array_values( $out );
    }
}
```

```php
    $blog_id = (int) $request->get_param( '_blog_id' );
    $body    = (string) $request->get_param( 'body' );
    $photo   = $_FILES['photo'] ?? null;

    $poll = null;
```

```php
private static function serialize_poll( int $post_id ): ?array {
    $options = \HeyFam\Core\Polls\Manager::options_for( $post_id );
    if ( ! $options ) return null;

    $count   = count( $options );
    $counts  = \HeyFam\Core\Polls\Manager::counts_for( $post_id, $count );
    $total   = array_sum( $counts );
    $closes  = (string) get_post_meta( $post_id, 'heyfam_poll_closes_at', true );
    $anon    = \HeyFam\Core\Polls\Manager::is_anon( $post_id );
    $closed  = \HeyFam\Core\Polls\Manager::is_closed( $post_id );
    $my_vote = \HeyFam\Core\Polls\Manager::my_vote( $post_id, get_current_user_id() );

    // Decorated option list — index, label, count, percent, is_my_vote.
    // Percent rounded to 0–100 ints. When total=0, all percents are 0.
    $option_rows = [];
    foreach ( $options as $i => $label ) {
        $n = (int) $counts[ $i ];
        $option_rows[] = [
            'index'      => $i,
            'label'      => $label,
            'count'      => $n,
            'percent'    => $total > 0 ? (int) round( ( $n / $total ) * 100 ) : 0,
            'is_my_vote' => $i === $my_vote,
            'bar_style'  => sprintf( 'width:%d%%;', $total > 0 ? (int) round( ( $n / $total ) * 100 ) : 0 ),
        ];
    }

    return [
        'options'      => $option_rows,
        'closes_at'    => $closes ? mysql2date( 'c', $closes, false ) : null,
        'closed'       => $closed,
        'anon'         => $anon,
        'total_votes'  => $total,
        'my_vote'      => $my_vote,
        'has_voted'    => $my_vote >= 0,
        'voters'       => $anon ? [] : \HeyFam\Core\Polls\Manager::voters_for( $post_id ),
    ];
}
```

```php
public function poll_vote( \WP_REST_Request $request ): \WP_REST_Response {
    $blog_id      = (int) $request->get_param( '_blog_id' );
    $post_id      = (int) $request->get_param( 'post_id' );
```

- The voter chips list iterates ALL voters and hides those not matching the current option's index. ## Open questions resolved

| Question (from orchestrator + brief) | Resolution |
|---|---|
| Custom post type vs `post_meta`? | `post_meta` on a regular post. Feed shape unchanged; post-card branches on `context.item.poll`. |
| Multi-choice? | Single-choice for v1. UNIQUE (post_id, user_id) enforces it. v2 can widen by adding `votes_per_user` meta and removing the constraint. |
| Compose UI shape? | Same-form toggle: "Make it a poll" swaps body fields for question + options. No modal. Photo button hides in poll mode (a poll already has its own visual hook). |
| Vote UI: hide pre-vote results? | No — always show. Empty bars before any votes; fill as votes land. The author wants the poll resolved; hiding adds friction. |
| Endpoint shape? | Extend `POST /{fam}/posts` for create (accepts `poll_options[]`, `poll_closes_at`, `poll_anon`). New `POST /{fam}/poll-vote` for casting. |
| Close-by date? | Optional, stored UTC. Server rejects votes once past; UI shows a "closed" badge. |
| Anonymous votes? | Per-poll toggle on composer. Default off (revealed). When on, voter chips suppressed and "anonymous" appended to meta line. |
| Reactions / comments on polls? | Yes, unchanged — polls are decorated posts. |
| Notifications on vote? | No. Posting a poll uses the existing post fan-out; voting is silent. |
| Change-vote allowed? | Yes — `INSERT … ON DUPLICATE KEY UPDATE` semantics. Tapping a different option just moves the vote. |
| SSR first paint? | Yes — `serialize_post` includes the poll decoration so the IAPI state injected via `wp_interactivity_state` in `functions.php` already carries `my_vote`, `results`, and `percents`. No hydration flicker. |
