# HeyFam Media Sharing — Multi-image v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let a fam member attach **multiple** photos to one post (drag-drop, paste, or file picker), render them in a tidy in-feed gallery (1 / 2 / 3 / 4+ layouts), and tap-to-open a lightbox. Backwards-compatible with today's single-photo posts. Video is deferred to v2.

**Architecture:**

- **Server.** A post's images live as WordPress attachments with `post_parent = $post_id`. The featured image (`_thumbnail_id`) remains set to the *first* attachment so the old `photo_url` field keeps working during migration. `Posts\Composer::create()` accepts an array of file slots (from a normalized `$_FILES['photos']`) and loops `media_handle_upload()` per slot. `Routes::serialize_post()` exposes both the legacy `photo_url` *and* a new `images: [{ id, url, thumb_url, alt, width, height }]` array; clients render whichever they prefer.
- **Client.** A new `heyfam/composer` store collects a `pendingFiles: File[]` list across the file-picker, a drop-zone, and a paste handler. Each file is shown as a thumbnail preview before submit. On submit, files are appended to one multipart `FormData` under `photos[]`. HEIC files are detected by extension/MIME; if the server has reported `heicSupport: true` (a tiny field on `wp_interactivity_state('heyfam')`), they're uploaded as-is; otherwise we lazy-`import('heic2any')` and convert in-browser before upload.
- **Gallery.** A new `parts/gallery.html` template part replaces the bare `<img>` inside `parts/post-card.html` and `templates/single.html`. CSS-only layout keyed off `data-count="1|2|3|4|many"`. Tapping any tile opens a 30-line vanilla lightbox.
- **Image sizes.** Register two custom sizes (`heyfam-feed-thumb` ~720w, `heyfam-feed-large` ~1600w) so we never serve full-resolution iPhone photos in the feed.

**Tech Stack:** PHP 8.1+, WordPress 6.5+ (HEIC server-side conversion is best-effort; needs 6.7+ to be automatic), `@wordpress/interactivity` (IAPI), esbuild. Optional dynamic import of `heic2any` (~430 KB minified, lazy-loaded only when needed).

**Depends on:** **design-system** (Phase B) — uses `heyfam-card`, `heyfam-button`, `heyfam-textarea`, `heyfam-input` classes from `components.css`; templates already wear those classes after design-system lands. **invite-onboarding** runs in Phase D before this — invite-onboarding only touches signup/invite templates, no overlap with composer/post-card. This plan runs **second** in Phase D, before polls.

**Rebase rule.** `serialize_post`, `parts/composer.html`, and `parts/post-card.html` are touched by design-system, this plan, and polls. The wholesale rewrites in earlier drafts of this plan have been converted into **additive Edits** that diff against the design-system version. Each Edit identifies the exact anchor in the file and patches only that hunk. Polls's plan applies a third pass on top of this one.

- [ ] **Step 4: Rewrite `Composer::create()` to accept N files**

Replace `plugins/heyfam-core/src/Posts/Composer.php` with:

```php
<?php
namespace HeyFam\Core\Posts;

final class Composer {
    /**
     * Create a post and attach zero or more uploaded photos.
     *
     * `$photos` accepts one of three shapes for backwards compatibility:
     *  - `null` or `[]`            — no photos.
     *  - `[ 'name'=>..., 'tmp_name'=>..., ... ]` — single $_FILES slot (legacy).
     *  - `[ [ 'name'=>..., ... ], [ 'name'=>..., ... ] ]` — list of slots (new).
     *
     * @return array{ post_id: int, attachment_ids: int[] }|\WP_Error
     */
    public static function create( int $author_id, string $body, $photos = null ): array|\WP_Error {
        $body  = trim( $body );
        $slots = self::normalize_slots( $photos );

        if ( $body === '' && empty( $slots ) ) {
            return new \WP_Error( 'empty_post', 'Add some text or a photo.' );
        }

        $post_id = wp_insert_post( [
            'post_type'    => 'post',
            'post_status'  => 'publish',
            'post_author'  => $author_id,
            'post_content' => wp_kses_post( $body ),
        ], true );
        if ( is_wp_error( $post_id ) ) return $post_id;

        $attachment_ids = [];
        if ( ! empty( $slots ) ) {
            require_once ABSPATH . 'wp-admin/includes/file.php';
            require_once ABSPATH . 'wp-admin/includes/media.php';
            require_once ABSPATH . 'wp-admin/includes/image.php';

            foreach ( $slots as $slot ) {
                if ( empty( $slot['tmp_name'] ) || ! empty( $slot['error'] ) ) continue;
                if ( ! self::is_acceptable_image( $slot ) ) continue;

                // media_handle_sideload accepts a slot-shaped array directly
                // and uploads from $slot['tmp_name'] without expecting $_FILES.
                $attach_id = media_handle_sideload( $slot, $post_id );
                if ( is_wp_error( $attach_id ) ) {
                    error_log( '[heyfam] photo upload failed: ' . $attach_id->get_error_message() );
                    continue;
                }
                $attachment_ids[] = (int) $attach_id;
            }

            if ( ! empty( $attachment_ids ) ) {
                // Legacy clients still read `photo_url` from `_thumbnail_id`.
                set_post_thumbnail( $post_id, $attachment_ids[0] );
            }
        }

        return [
            'post_id'        => (int) $post_id,
            'attachment_ids' => $attachment_ids,
        ];
    }

    /** Coerces the three input shapes into a list of $_FILES-style slots. */
    private static function normalize_slots( $photos ): array {
        if ( empty( $photos ) ) return [];
        // Legacy single-slot shape: `tmp_name` is a string at the top level.
        if ( isset( $photos['tmp_name'] ) && is_string( $photos['tmp_name'] ) ) {
            return [ $photos ];
        }
        // List of slots — values are themselves arrays with `tmp_name` strings.
        $out = [];
        foreach ( $photos as $slot ) {
            if ( is_array( $slot ) && isset( $slot['tmp_name'] ) && is_string( $slot['tmp_name'] ) ) {
                $out[] = $slot;
            }
        }
        return $out;
    }

    /** Allow only browser-safe images. Server-side HEIC→JPEG is handled in core 6.7+. */
    private static function is_acceptable_image( array $slot ): bool {
        $allowed = [ 'image/jpeg', 'image/png', 'image/gif', 'image/webp', 'image/avif', 'image/heic', 'image/heif' ];
        $mime    = $slot['type'] ?? '';
        if ( in_array( $mime, $allowed, true ) ) return true;
        // Fall back to extension check — Safari quirkily reports HEIC as octet-stream.
        $ext = strtolower( pathinfo( $slot['name'] ?? '', PATHINFO_EXTENSION ) );
        return in_array( $ext, [ 'jpg', 'jpeg', 'png', 'gif', 'webp', 'avif', 'heic', 'heif' ], true );
    }
}
```

Notes on the swap from `media_handle_upload` → `media_handle_sideload`: both create an attachment, but `media_handle_upload` reads from `$_FILES[$key]` directly. With a list, normalizing to one canonical slot per file and passing it to `media_handle_sideload` is cleaner than rebuilding `$_FILES` per iteration. Sideload moves the temp file (`is_uploaded_file()` is bypassed), so the caller must ensure files came from a verified `$_FILES` (we do — see Task 1 Step 5).

```php
public function create_post( \WP_REST_Request $request ): \WP_REST_Response {
    $blog_id = (int) $request->get_param( '_blog_id' );
    $body    = (string) $request->get_param( 'body' );

    // Prefer the new `photos[]` shape; fall back to the legacy single `photo`.
    $slots = self::normalize_files_array( $_FILES['photos'] ?? null );
    if ( empty( $slots ) && isset( $_FILES['photo'] ) ) {
        $slots = [ $_FILES['photo'] ];
    }

    // Cap to a sane limit so a thumb fat-finger doesn't upload an album.
    $slots = array_slice( $slots, 0, 10 );

    $result = \HeyFam\Core\Auth\Authorization::in_blog( $blog_id, function () use ( $body, $slots ) {
        return \HeyFam\Core\Posts\Composer::create( get_current_user_id(), $body, $slots );
    } );
    if ( is_wp_error( $result ) ) {
        return new \WP_REST_Response(
            [ 'error' => $result->get_error_code(), 'message' => $result->get_error_message() ],
            400
        );
    }
    return new \WP_REST_Response( [
        'ok'              => true,
        'post_id'         => $result['post_id'],
        'attachment_ids'  => $result['attachment_ids'],
    ], 201 );
}

/**
 * Turn the awkward $_FILES['photos'] shape:
 *   [ 'name' => [a,b], 'tmp_name' => [a,b], ... ]
 * into a list of per-file slots:
 *   [ [ 'name'=>a, 'tmp_name'=>a, ... ], [ 'name'=>b, ... ] ]
 */
private static function normalize_files_array( ?array $files ): array {
    if ( empty( $files ) || ! is_array( $files['name'] ?? null ) ) return [];
    $count = count( $files['name'] );
    $out   = [];
    for ( $i = 0; $i < $count; $i++ ) {
        if ( ! empty( $files['error'][ $i ] ) ) continue;
        $out[] = [
            'name'     => $files['name'][ $i ]     ?? '',
            'type'     => $files['type'][ $i ]     ?? '',
            'tmp_name' => $files['tmp_name'][ $i ] ?? '',
            'error'    => $files['error'][ $i ]    ?? 0,
            'size'     => $files['size'][ $i ]     ?? 0,
        ];
    }
    return $out;
}
```

- [ ] **Step 1: Add a helper that builds the image list for a post**

```php
/**
 * All attachments hung off this post, ordered by the WP `menu_order` (which
 * matches upload order by default) then ID. Returns one entry per image
 * with both feed and lightbox URLs.
 *
 * @return array<int, array{ id:int, url:string, thumb_url:string, alt:string, width:int, height:int }>
 */
private static function images_for_post( int $post_id ): array {
    $attachments = get_children( [
        'post_parent'    => $post_id,
        'post_type'      => 'attachment',
        'post_mime_type' => 'image',
        'orderby'        => [ 'menu_order' => 'ASC', 'ID' => 'ASC' ],
        'order'          => 'ASC',
        'numberposts'    => 20,
    ] );
    if ( ! $attachments ) return [];

    $out = [];
    foreach ( $attachments as $att ) {
        $id        = (int) $att->ID;
        $large     = wp_get_attachment_image_src( $id, 'heyfam-feed-large' );
        $thumb     = wp_get_attachment_image_src( $id, 'heyfam-feed-thumb' );
        $full      = wp_get_attachment_image_src( $id, 'full' );
        $url       = $large[0] ?? $full[0] ?? '';
        $thumb_url = $thumb[0] ?? $url;
        if ( ! $url ) continue;
        $out[] = [
            'id'        => $id,
            'url'       => $url,
            'thumb_url' => $thumb_url,
            'alt'       => (string) get_post_meta( $id, '_wp_attachment_image_alt', true ),
            'width'     => (int) ( $large[1] ?? $full[1] ?? 0 ),
            'height'    => (int) ( $large[2] ?? $full[2] ?? 0 ),
        ];
    }
    return $out;
}
```

Replace the entire contents of `themes/heyfam-theme/src/interactivity/composer.js` with:

```js
import { store, getContext } from '@wordpress/interactivity';

const MAX_FILES   = 10;
const MAX_BYTES   = 25 * 1024 * 1024; // 25 MB per file before client conversion
const ACCEPT_RE   = /^image\/(jpeg|png|gif|webp|avif|heic|heif)$/i;
const HEIC_EXT_RE = /\.(heic|heif)$/i;

const { state, actions } = store( 'heyfam/composer', {
  state: {
    body: '',
    /**
     * Array of { id, file, name, previewUrl, status } where status is
     * 'pending' | 'converting' | 'ready' | 'error'. Previews are
     * `URL.createObjectURL` strings; we revoke them on remove + on submit.
     */
    pending: [],
    submitting: false,
    error: '',
    get hasFiles() { return this.pending.length > 0; },
    get canSubmit() {
      // Allow text-only posts (body trimmed), or any number of ready files.
      return ! this.submitting && ( this.body.trim() !== '' || this.pending.some( p => p.status === 'ready' ) );
    },
    get photoLabel() {
      return this.pending.length === 0 ? 'Add photos' : `${this.pending.length} photo${ this.pending.length === 1 ? '' : 's' }`;
    },
  },
  actions: {
    updateBody( e ) { state.body = e.target.value; },

    /** Triggered by the <input type="file" multiple> change event. */
    pickPhotos( e ) {
      const files = Array.from( e.target.files || [] );
      addFiles( files );
      // Clear the input so picking the same filename again still fires `change`.
      e.target.value = '';
    },

    /** Drag-drop on the composer's drop-zone. */
    onDragOver( e ) {
      if ( e.dataTransfer?.types?.includes( 'Files' ) ) {
        e.preventDefault();
        e.currentTarget.classList.add( 'is-drop-target' );
      }
    },
    onDragLeave( e ) {
      e.currentTarget.classList.remove( 'is-drop-target' );
    },
    onDrop( e ) {
      e.preventDefault();
      e.currentTarget.classList.remove( 'is-drop-target' );
      const files = Array.from( e.dataTransfer?.files || [] );
      addFiles( files );
    },

    /** Cmd/Ctrl+V on the body textarea — grab image clipboard items. */
    onPaste( e ) {
      const items = Array.from( e.clipboardData?.items || [] );
      const files = items
        .filter( i => i.kind === 'file' )
        .map( i => i.getAsFile() )
        .filter( Boolean );
      if ( files.length === 0 ) return; // text paste: let the textarea handle it
      e.preventDefault();
      addFiles( files );
    },

    removeFile() {
      const ctx = getContext();
      const id  = ctx?.item?.id;
      const idx = state.pending.findIndex( p => p.id === id );
      if ( idx === -1 ) return;
      const removed = state.pending[ idx ];
      if ( removed.previewUrl ) URL.revokeObjectURL( removed.previewUrl );
      state.pending.splice( idx, 1 );
    },

    *submit( e ) {
      e.preventDefault();
      if ( ! state.canSubmit || state.submitting ) return;
      state.submitting = true;
      state.error      = '';
      const heyfam    = store( 'heyfam' ).state;

      const ready = state.pending.filter( p => p.status === 'ready' );
      const fd    = new FormData();
      fd.append( 'body', state.body );
      for ( const p of ready ) fd.append( 'photos[]', p.file, p.name );

      try {
        const r = yield fetch( `${heyfam.apiBase}/${heyfam.famSlug}/posts`, {
          method: 'POST', credentials: 'include',
          headers: { 'X-WP-Nonce': heyfam.nonce }, body: fd,
        } );
        if ( ! r.ok ) throw new Error( 'post-failed' );

        // Clear state, revoke object URLs.
        state.body = '';
        for ( const p of state.pending ) p.previewUrl && URL.revokeObjectURL( p.previewUrl );
        state.pending = [];

        store( 'heyfam/feed' ).callbacks.refresh( heyfam );
      } catch ( err ) {
        state.error = 'Could not post. Try again.';
      } finally {
        state.submitting = false;
      }
    },
  },
} );

function addFiles( files ) {
  let nextId = ( state.pending[ state.pending.length - 1 ]?.id ?? 0 ) + 1;
  for ( const file of files ) {
    if ( state.pending.length >= MAX_FILES ) {
      state.error = `Max ${MAX_FILES} photos per post.`;
      break;
    }
    if ( ! acceptable( file ) ) {
      state.error = 'Only images, please.';
      continue;
    }
    if ( file.size > MAX_BYTES ) {
      state.error = `${file.name} is too large.`;
      continue;
    }

    const slot = {
      id:         nextId++,
      file,
      name:       file.name,
      previewUrl: URL.createObjectURL( file ),
      status:     'pending',
    };
    state.pending.push( slot );

    // HEIC handoff is Task 4. For now everything is immediately 'ready'.
    slot.status = 'ready';
  }
}

function acceptable( file ) {
  return ACCEPT_RE.test( file.type ) || HEIC_EXT_RE.test( file.name );
}
```

Important: the IAPI proxies arrays — `splice`, `push`, etc. work. `URL.createObjectURL` is fine for `<img src>` bindings via `data-wp-bind--src`.

async function addFiles( files ) {
  const heyfam = store( 'heyfam' ).state;
  let nextId   = ( state.pending[ state.pending.length - 1 ]?.id ?? 0 ) + 1;

  for ( const file of files ) {
    if ( state.pending.length >= MAX_FILES ) { state.error = `Max ${MAX_FILES} photos per post.`; break; }
    if ( ! acceptable( file ) )              { state.error = 'Only images, please.'; continue; }
    if ( file.size > MAX_BYTES )             { state.error = `${file.name} is too large.`; continue; }

    const isHeic = HEIC_EXT_RE.test( file.name ) || /heif|heic/i.test( file.type );
    const needsClientConvert = isHeic && ! heyfam.heicSupport;

    const slot = {
      id:         nextId++,
      file,
      name:       file.name,
      previewUrl: '', // set below
      status:     needsClientConvert ? 'converting' : 'ready',
    };
    // Provisional preview from the original file (browsers can render
    // some HEIC into <img> on iOS; if it fails, the slot still posts).
    try { slot.previewUrl = URL.createObjectURL( file ); } catch {}
    state.pending.push( slot );

    if ( needsClientConvert ) {
      try {
        const { default: heic2any } = await import( /* @vite-ignore */ 'heic2any' );
        const blob = await heic2any( { blob: file, toType: 'image/jpeg', quality: 0.82 } );
        const out  = Array.isArray( blob ) ? blob[0] : blob;
        const converted = new File(
          [ out ],
          file.name.replace( /\.(heic|heif)$/i, '.jpg' ),
          { type: 'image/jpeg' }
        );
        // Swap the slot's file + preview.
        const fresh = state.pending.find( p => p.id === slot.id );
        if ( ! fresh ) return; // user removed it during conversion
        if ( fresh.previewUrl ) URL.revokeObjectURL( fresh.previewUrl );
        fresh.file       = converted;
        fresh.name       = converted.name;
        fresh.previewUrl = URL.createObjectURL( converted );
        fresh.status     = 'ready';
      } catch ( err ) {
        const fresh = state.pending.find( p => p.id === slot.id );
        if ( fresh ) fresh.status = 'error';
        state.error = 'Could not read HEIC photo. Convert to JPEG and try again.';
      }
    }
  }
}

```js
import { store, getContext } from '@wordpress/interactivity';

const { state, actions } = store( 'heyfam/lightbox', {
  state: {
    open:     false,
    images:   [],  // full-res URLs for the currently-opened post
    index:    0,
    /** Computed accessors used by template directives. */
    get current()  { return this.images[ this.index ] || null; },
    get currentUrl() { return this.current?.url || ''; },
    get currentAlt() { return this.current?.alt || ''; },
    get hasPrev()  { return this.index > 0; },
    get hasNext()  { return this.index < this.images.length - 1; },
    get position() {
      return this.images.length > 1
        ? `${this.index + 1} / ${this.images.length}`
        : '';
    },
  },
  actions: {
    open( e ) {
      e.preventDefault();
      const ctx       = getContext();
      // `context.item` is the post (set by data-wp-each in the post-card),
      // `context.photo` is the specific image clicked.
      const post      = ctx?.item;
      const photo     = ctx?.photo;
      if ( ! post || ! photo ) return;
      state.images = ( post.images || [] ).map( i => ( { id: i.id, url: i.url, alt: i.alt } ) );
      state.index  = state.images.findIndex( i => i.id === photo.id );
      if ( state.index < 0 ) state.index = 0;
      state.open   = true;
      document.body.classList.add( 'heyfam-lightbox-open' );
    },
    close() {
      state.open = false;
      document.body.classList.remove( 'heyfam-lightbox-open' );
    },
    prev() { if ( state.hasPrev ) state.index--; },
    next() { if ( state.hasNext ) state.index++; },
    onBackdrop( e ) {
      if ( e.target.classList.contains( 'heyfam-lightbox' ) ) actions.close();
    },
    onKey( e ) {
      if ( ! state.open ) return;
      if ( e.key === 'Escape' )   actions.close();
      if ( e.key === 'ArrowLeft' ) actions.prev();
      if ( e.key === 'ArrowRight') actions.next();
    },
  },
  callbacks: {
    // Run once at hydration: bind global key handler.
    init() {
      document.addEventListener( 'keydown', actions.onKey );
    },
  },
} );
```

## Open questions resolved

- **Storage strategy:** WP media library, attachments hung off the post via `post_parent = $post_id`. We loop `media_handle_sideload()` per file. The first attachment is also the post's `_thumbnail_id` so the legacy `photo_url` field continues to work without a data migration.
- **HEIC / iOS photos:** Server-first (WP 6.7+ + Imagick libheif handles this automatically). When the server reports no HEIC support, the composer lazy-loads `heic2any` (dynamic import, code-split chunk ~430 KB, only paid for by users who actually post HEIC on hosts without delegate support). Doesn't punt; doesn't bloat the main bundle.
- **Image resize/optimize:** Two registered sizes — `heyfam-feed-thumb` (720) for tiles, `heyfam-feed-large` (1600) for lightbox. Falls back to `full` if registration hasn't run yet. WP auto-generates the variants on upload via its existing intermediate-size pipeline.
- **Gallery layout:** Pure CSS grid keyed off `data-count="1|2|3|4|many"`. No JS layout math, no library. 1 = contain-fit hero, 2 = 1×2, 3 = 1-left/2-right, 4 = 2×2, 5+ = 2×2 with `+N` badge.
- **Lightbox:** Hand-rolled, ~60 lines of JS, no dependency. Esc / arrow / swipe / backdrop close. `basicLightbox` would be fine too but offers ~nothing we'd actually use.
- **Video:** Deferred to v2. Video upload (`media_handle_sideload` works) is easy, but playable cross-device playback wants HLS / transcoding (server-side ffmpeg) and a `<video>` thumbnail strip. That's days of work and out of scope here. v2 plan should cover: max-length cap (e.g., 30s), client-side downscale via WebCodecs *or* server-side `wp ffmpeg` job, MP4-only delivery to all browsers (no HLS) for v2 simplicity.
- **REST + IAPI integration:** `serialize_post` returns both `photo_url` (legacy, equals first image's `heyfam-feed-large` URL) and a new `images: [{id, url, thumb_url, alt, width, height}]` array, plus `image_count`. The post-card template iterates `images[]` via a nested `data-wp-each` inside the gallery template part. `data-wp-bind--data-count` drives the CSS layout.
- **Per-post cap & per-fam storage cap:** Per-post hard cap of 10 photos (server `array_slice` and client `MAX_FILES`). Per-fam storage cap deferred — track in v2 (orchestrator question).
- **Build step:** No new top-level build step. esbuild handles `heic2any` via dynamic `import()` — add `splitting: true` to `scripts/build-theme.mjs` so the chunk is split out instead of inlined into `index.js`.
